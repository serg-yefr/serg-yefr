BaaS для партнеров : Отмена списания / hold_reverse

Created by Наталья Боровикова, last modified by Наталья Моргунова on февр. 04, 2022

Метод используется для отмены холдирования. После отмены денежные средства моментально возвращаются на карту клиента.

Можно отменить холд на сумму частично. Оставшаяся часть останется захолдирована.

POST /api/v1/hold/reverse/{order_slug}

URL параметры

Описание поля

Тип

order_slug

Уникальный идентификатор операции (не более 40 символов)

string (path)

POST параметры

Описание поля

Тип

amount

Сумма частичной отмены холдирования (обычно не указывается)

int (nullable)

Пример ответа: Status 200 OK

{
  "order_slug": "client_wof_32425_8e4Q0WyK",
  "amount": 5000
}

Пример ответа при ошибке статуса: Status 400 Bad Request

{
    "description": "Can't confirm hold with status error"
}

Пример ответа ошибки токена: Status 403 Forbidden

{
    "description": "Forbidden",
    "errors": [],
    "date": "2021-10-11T08:23:48+00:00"
}

Пример ответа при недостатке прав: Status 403 Forbidden

{
    "description": "Access Denied."
}

Пример ответа при ошибочном номере карты: Status 404 Not found

{
    "description": "Requested object is not found"
}

Пример ответа при ошибке метода: Status 500 Internal Server Error

{
    "description": "Runtime error. Please try later."
}

 

Поле ответа

Описание поля

Тип

amount

Сумма частичной отмены холдирования

int (nullable)

order_slug

Уникальный идентификатор операции со стороны партнера (не более 40 символов)

string
