BaaS для партнеров : Сценарий взаимодействия

Created by Vladimir Chesnokov on окт. 12, 2021

Партнер - это маркетплейс или платформа, которая осуществляет взаимодействие между исполнителями и заказчиками или продавцами и покупателями.

Бенефициар - это компания/физ.лицо, которому принадлежат средства на счете.

Управление бенефициарами используется в кейсах, когда на едином счете хранятся и управляются средства различных заказчиков/мерчантов/исполнителей (бенефициаров).

Алгоритм работы:

  1. Партнер создает сущность бенефициара с помощью метода 10.1 beneficiary_add

    1. В запросе передается название и ИНН бенефициара.

    2. В ответ Партнер получает идентификатор бенефициара, который используется в дальнейшем взаимодействии.

    3. Идентификатор бенефициара необходимо передавать при каждом запросе выплаты, для корректного учета баланса

  2. Для редактирования или блокировки бенефициара используется метод 10.2 beneficiary_edit

  3. Для создания комиссии, которую Партнер взимает с бенефициара используется метод 10.5 beneficiary_commission_add

    1. Комиссия рассчитывается за каждую проведенную выплату

    2. Дата начала действия комиссии - это дата с которой комиссия вступает в силу

    3. Чтобы изменить комиссию по бенефициару необходимо создать новую комиссию с датой вступления изменений.

      1. Партнер создал комиссию, которая действует с 01.10.2020 в размере 3%

      2. Партнер создает 2ю комиссию с датой начала действия от 01.11.2020 в размере 2%

      3. С 01.11.2020 с бенефициара начнет взиматься новая комиссия 2%

    4. Типы комиссий: % от суммы операции, % от суммы операции ПЛЮС фикс сумма, % от суммы операции не менее фикс суммы

      1. 3%

      2. 3% + 2 руб. за операцию

      3. 3%, но не менее 30 руб. за операцию

  4. Для редактирования комиссии используется метод 10.6 beneficiary_commission_edit

  5. Для удаления комиссии используется метод 10.7 beneficiary_commission_delete

  6. Для получения списка созданных комиссий по бенефициару используется метод 10.8 beneficiary_commission_list

  7. Для получения информации о бенефициаре, его актуальном остатке средств и начисленной комиссии используется метод 10.3 beneficiary_show
