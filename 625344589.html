BaaS для партнеров : Авторизация карты без формы / payment_authorization

Created by Наталья Боровикова, last modified by Наталья Моргунова on янв. 25, 2022

Метод используется для сохранения реквизитов карты стороннего банка в системе.

Актуален для партнеров, имеющих сертификат PCI DSS. В случае отсутствия сертификата PCI DSS для авторизации следует использовать метод Авторизации карты с формой.

Описание процесса авторизации:

Клиент оставляет данные карты в сервисе партнера. При вызове метода партнер передает реквизиты карты, сохраненные на своей стороне, а в ответ получает ссылку на страница для подтверждения операции кодом 3ds. Партнер переадресовывает клиента на полученную форму, где клиент подтверждает операцию.

При регистрации с карты будет списана авторизационная сумма, а затем автоматически возвращена.

Для получения идентификатора карты в системе необходимо вызвать метод Получение статуса платежа / payment_status. Идентификатор карты будет указан в поле сard_token.

Идентификатор необходимо сохранить для использования в дальнейших операциях.

Финальный статус авторизационного платежа - cancelled.

Время жизни ссылки  - 15 минут.
