BaaS для партнеров : Списание с авторизованной карты без 3dsecure / payment_from_registered_card

Created by Наталья Боровикова, last modified by Наталья Моргунова on февр. 04, 2022

Метод используется для рекурентных платежей с карты клиента. Например, досписание оплаты за услугу, подписки на сервис и т.д.

Метод может быть использован при условии, что карта ранее была авторизована. Для авторизации карты используются методы Авторизация клиентов

На счет партнера поступит сумма, указанная в поле "amount" за вычетом комиссии Talkbank.

Для уточнения статуса платежа используется метод Получение статуса платежа / payment_status

POST /api/v1/payment/from/{client_id}/registered/card

URL параметры

Описание поля

Тип

client_id

Уникальный идентификатор клиента на стороне партнера

string (path)

POST параметры

Описание поля

Тип

amount

Сумма списания в копейках

int

order_slug

Уникальный идентификатор операции со стороны партнера (не более 40 символов)

string

card_token

Уникальный идентификатор карты в формате uuid

string

Пример запроса

{
    "amount": 100,
    "card_token": "7daf4b05-2860-4e57-b1db-bccd40840bc5",
	"order_slug":"client_wof_32425_8e4Q0WyK"
}

Пример ответа: Status 200 OK

{
	"order_slug": "client_wof_32425_8e4Q0WyK"
}

Пример ответа при ошибке card_token: Status 400 Bad Request

{
    "description": "Can't confirm hold with status error"
}

Пример ответа ошибки токена: Status 403 Forbidden

{
    "description": "Forbidden",
    "errors": [],
    "date": "2021-10-11T08:23:48+00:00"
}

Пример ответа при недостатке прав: Status 403 Forbidden

{
    "description": "Access Denied."
}

Пример ответа при ошибочном номере карты: Status 404 Not found

{
    "description": "Requested object is not found"
}

Пример ответа при ошибке метода: Status 500 Internal Server Error

{
    "description": "Runtime error. Please try later."
}
