BaaS для партнеров : Получение статуса клиента / client_show

Created by Наталья Боровикова, last modified on дек. 25, 2020

Метод позволяет получить статус идентификации клиента.

GET /api/v1/clients/{client_id}

URL параметры

Описание

Тип

client_id

Идентификатор клиента

string (path)

Пример ответа

{
  "client_id": "123154",
  "is_identified": false,
  "status": "Identification in process",
  "status_code": "error",
  "message": "passport: doc_status = 302: сведениями по заданным реквизитам не располагаем",
  "additional": {
    "inn": {
      "status": "Processing",
      "description": "В обработке",
    }
    "snils": {
      "status": "NotProcessed",
      "description": "Не указан СНИЛС, проверка не производилась"
    }
    "passport": {
      "status": "NotValid",
      "description": "DOC_STATUS = 302: Сведениями по заданным реквизитам не располагаем"
    }
  }
}

 

Поле ответа

Описание поля

client_id

Идентификатор клиента

is_identified

Статус идентификации true/false

status

Статус идентификации (Identification in process|Identification complete|Identification Error)

status_code

error|success|processing

message

расшифровка ошибки (свободная форма)

additional

расшифровка по полям (см. событие identitifcaion.error)
