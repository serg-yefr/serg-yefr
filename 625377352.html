BaaS для партнеров : Получение токена для авторизации незарегистрированной карты / payment_authorization_token

Created by Наталья Боровикова, last modified by Наталья Моргунова on янв. 25, 2022

POST /api/v1/authorize/{clientId}/token

POST параметры

Описание поля

Тип

redirect_url

Ссылка на страницу, на которую попадет пользователь после ввода 3dsec

string

Пример запроса

{
    "redirect_url": "https://example.com",
	"amount": 100
}

Пример ответа: Status 200 ОК

{
    "token": "df9a1ab1-6928-4a67-8f28-cb465510bc03",
    "token_status": "eec3e78f-1698-458f-b19f-879873aaac38",
    "order_id": 52773,
    "order_slug": "client_wof_35545_yb3xwEMD"
}

Пример ответа ошибки токена: Status 403 Forbidden

{
    "description": "Forbidden",
    "errors": [],
    "date": "2021-10-11T08:23:48+00:00"
}

Пример ответа при несуществующем баркоде: Status 404 Not Found

{
    "description": "Requested object is not found"
}

Пример ответа при ошибке метода: Status 500 Internal Server Error

{
    "description": "Runtime error. Please try later."
}

Поле ответа

Описание поля

Тип

token

Токен, необходимый для запроса к публичному методу авторизации карты. Время жизни - 15 минут. Количество попыток - 1

string

order_id

Уникальный идентификатор операции

int

token_status

Токен, необходимый для запроса к публичному методу получения статуса операции

string

order_slug

Уникальный идентификатор операции на стороне клиента

string
