BaaS для партнеров : Холдирование без формы / hold

Created by Наталья Боровикова, last modified by Наталья Моргунова on февр. 02, 2022

Метод холдирует указанную сумму на карте. Сбор карточных данных происходит на стороне партнера (при наличии сертификата PCI DSS). В ответ на запрос метод вернет ссылку на подтверждение платежа кодом из смс.

POST /api/v1/hold

POST параметры

Описание поля

Тип

amount

Сумма списания в копейках

int

order_slug

Уникальный идентификатор операции на стороне партнера (не более 40 символов)

string

card_info.number

Номер карты (либо card_token)

string

card_info.month

Месяц окончания срока действия карты

string

card_info.year

Год окончания срока действия карты

string

card_info.cvv

Cvv/cvc код карты

string

card_info.holder

Держатель карты

string

card_token

Уникальный идентификатор карты (либо card_info.*)

string

redirect_url

Ссылка на страницу, на которую попадет пользователь после ввода 3dsec

string

Пример запроса

{
    "amount": 100,
	"card_token": "assaffgqw12334"
    "redirect_url": "https://example.com",
	"order_slug":"client_wof_32425_8e4Q0WyK"
}

Пример ответа : Status 200 OK

{
  "hold_link": "https://baas.talkbank.io/charge/3",
  "order_id": 3,
  "order_slug": "client_wof_32425_8e4Q0WyK"
}

Пример ответа ошибки токена: Status 403 Forbidden

{
    "description": "Forbidden",
    "errors": [],
    "date": "2021-10-11T08:23:48+00:00"
}

Пример ответа при недостатке прав: Status 403 Forbidden

{
    "description": "Access Denied."
}

Пример ответа при ошибочном номере карты: Status 404 Not found

{
    "description": "Requested object is not found"
}

Пример ответа при ошибке метода: Status 500 Internal Server Error

{
    "description": "Runtime error. Please try later."
}

Поле ответа

Описание поля

charge_link

Ссылка для перехода на ввод 3dsec

order_id

Уникальный идентификатор операции
