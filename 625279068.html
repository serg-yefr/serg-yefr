BaaS для партнеров : Получение транзакций по всем картам партнера / account_cards_transactions

Created by Наталья Боровикова, last modified by Наталья Моргунова on февр. 04, 2022

Метод позволяет партнеру получить историю транзакций по карте клиента.

Расходные операции будут отражаться со знаком "-"

GET /api/v1/cards-transactions

URL Параметры

Описание

По умолчанию

Тип

fromDate

С какой даты показать историю транзакций

-1 month

ISOdate (query)

toDate

По какую дату показать историю транзакций

now

ISOdate (query)

limit

Лимит записей, максимум 1000

1000

int (query)

page

Страница

1

int (query)


Пример ответа: Status 200 OK

{
    "data": []
}

Пример ответа ошибки токена: Status 403 Forbidden

{
    "description": "Forbidden",
    "errors": [],
    "date": "2021-10-11T08:23:48+00:00"
}

Пример ответа при недостатке прав: Status 403 Forbidden

{
    "description": "Access Denied."
}

Пример ответа при ошибочном номере карты: Status 404 Not found

{
    "description": "Requested object is not found"
}

Пример ответа при ошибке метода: Status 500 Internal Server Error

{
    "description": "Runtime error. Please try later."
}

Расшифровка полей транзакции

Поле ответа

Описание поля

Примечание

id

Идентификационный номер

Для холдированных операций равен "0", после подтверждения присваивается уникальный id

datetime

Дата и время совершения операции

 

cardNumber

Номер карты, с которой была совершена транзакция

Выводятся первые и последние четыре цифры карты

amount

Сумма в валюте транзакции

 

cardAmount

Сумма транзакции в валюте карты

 

hold

Показывает, что операция «заморожена»

Для холдированных операций – true, для подтвержденных – поле отсутствует

type

Тип транзакции

debit – расходная, credit - доходная

currency

Валюта транзакции

 

category.parent

Родительская категория

category.name

Имя категории

mcc

MMC-код торговой точки

 

barcode

баркод карты
