BaaS для партнеров : Авторизация карты с формой / payment_authorization_with_form

Created by Наталья Боровикова, last modified by Наталья Моргунова on февр. 04, 2022

Метод используется для сохранения реквизитов карты стороннего банка в системе.

Предназначен для партнеров, не имеющих сертификат безопасности PCI DSS, т.к. не предполагает хранение карточных данных в их система.

Описание процесса авторизации:

Партнер вызывает метод, в ответ получает ссылку, на которую переадресовывает клиента.

Клиент по ссылке вводит реквизиты карты и подтверждает операцию.

При регистрации с карты будет списана авторизационная сумма, а затем автоматически возвращена.

У клиента есть только одна попытка ввести данные при переходе по ссылке. При ошибке авторизации, необходимо вызвать метод еще раз и передать клиенту новую ссылку.

Для получения идентификатора карты в системе необходимо вызвать метод Получение статуса платежа / payment_status. Идентификатор карты будет указан в поле сard_token.

Идентификатор необходимо сохранить для использования в дальнейших операциях.

Финальный статус авторизационного платежа - cancelled

Время жизни ссылки - 15 минут.

Пример запроса

{
    "redirect_url": "https://example.com",
	"order_slug":"client_wof_32425_8e4Q0WyK"
}

Пример ответа: Status 200 ОК

{
  "charge_link": "https://baas.talkbank.io/charge/3",
  "order_id": 3,
  "order_slug": "client_wof_32425_8e4Q0WyK"
}

Пример ответа ошибки токена: Status 403 Forbidden

{
    "description": "Forbidden",
    "errors": [],
    "date": "2021-10-11T08:23:48+00:00"
}

Пример ответа при несуществующем client _id: Status 404 Not Found

{
    "description": "Requested object is not found"
}

Пример ответа при несуществующем order_slug: Status 422 Unprocessable Entity

{
    "description": "order_slug must be [\\w.:-]{10,38}",
    "errors": []
}

Пример ответа при ошибке метода: Status 500 Internal Server Error

{
    "description": "Runtime error. Please try later."
}
