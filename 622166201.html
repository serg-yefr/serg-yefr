BaaS для партнеров : Пополнение карты Talkbank со счета партнера / card_refill

Created by Наталья Боровикова, last modified by Игорь Бекренёв on февр. 21, 2022

Метод используется для пополнения карт Talkbank, выпущенных партнером.

На карту клиента будет зачислена сумма, указанная в поле amount.

Комиссия за транзакцию будет списана со счета партнера.

Для запроса статуса платежа следует использовать метод Получение статуса платежа / payment_status

POST /api/v1/clients/{client_id}/cards/{barcode}/refill

Параметры

Описание

Тип

barcode

Баркод карты

string (path)

amount

Сумма пополнения в копейках

int

order_slug

Уникальный идентификатор операции со стороны партнера (не более 40 символов)

string

percents_on_user

Переложить комиссии на клиента

boolean

receipt_id

Идентификатор чека самозанятого

string

receipt_ids

Идентификаторы чеков самозанятого

array

beneficiary_id

Идентификатор бенефициара

string

Пример входных параметров

Пример запроса

{
    "amount": 100,
	"order_slug":"client_wof_32425_8e4Q0WyK",
	"percents_on_user":"0"
}

Пример ответа

{
	"order_id": 3,
	"order_slug": "client_wof_32425_8e4Q0WyK"
}

Пример ответа ошибки токена: Status 403 Forbidden

{
    "description": "Forbidden",
    "errors": [],
    "date": "2021-10-11T08:23:48+00:00"
}

Пример ответа при недостатке прав: Status 403 Forbidden

{
    "description": "Access Denied."
}

Пример ответа при ошибочном barcode: Status 404 Not found

{
    "description": "Requested object is not found"
}

Пример ответа при запросе с избыточным параметром: Status 422 Unprocessable Entity

{
    "description": "Validation error list",
    "errors": [
        "[barcode]: Это поле не ожидалось."
    ]
}

Пример ответа при ошибке метода: Status 500 Internal Server Error

{
    "description": "Runtime error. Please try later."
}

int

order_slug
