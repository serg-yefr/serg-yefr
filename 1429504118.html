BaaS для партнеров : Получение платежной квитанции / payment_receipt

Created by Игорь Бекренёв, last modified on янв. 28, 2022

Метод может быть использован для получения ссылки на платёжную квитанцию по операциям:

  • charge_auth

  • charge_hold

  • charge_registered

  • charge_unregistered

  • withdrawal

  • refill_registered

  • refill_unregistered

  • refill

GET /api/v1/payment/{order_slug}/receipt

URL параметры

Описание поля

Тип

order_slug

Уникальный идентификатор операции со стороны партнера

string (path)

Пример ответа: Status 200 OK

{
  "link": "https:\/\/storage.yandexcloud.net\/baas-payment-receipts\/017bd034-32d0-fb18-502b-b46144a6ef79.pdf"
}

Пример ответа ошибки токена: Status 403 Forbidden

{
    "description": "Forbidden",
    "errors": [],
    "date": "2021-10-11T08:23:48+00:00"
}

Пример ответа при несуществующей платежной квитанции: Status 404 Not Found

{
    "description": "Requested object is not found"
}

Пример ответа при ошибке метода: Status 500 Internal Server Error

{
    "description": "Runtime error. Please try later."
}
