BaaS для партнеров : Оформление подписки / event_subscription_store

Created by Наталья Боровикова, last modified by Алексей Кузьмин on нояб. 26, 2021

Позволяет получать уведомления о выбранных событиях на переданный URL. Возможно указание разных URL для разных событий, а так же более одного URL на каждое событие.

POST /api/v1/event-subscriptions

POST параметры

Описание поля

Допустимы значения

Примечание

url

Адрес, куда будут уходить события

Корректный URL

Обязателен

events[]

Множество типов событий

"payment",
"delivery",
"identification",
"activation",
"unsigned_charge",
"order",
"special_uralsib_cards",
"selfemployed_status",
"income",
"transfer"

Пустой массив или отсутствующее значение параметра - означает все типы

Пример входных параметров

{
    "url": "https://example.com/webhook",
    "events": ["payment", "activation"]
}

После успешного удаления возвращает список event_subscription_list

  • Нотификации о расходах (notifications)

  • Изменение статуса доставки (delivery)

  • Изменение статуса идентификации (identification)

  • Изменение статуса активации (activation)

  • Изменение статуса банковского ордера (order)

  • Изменение статуса регистрации самозанятого (selfemployments)

  • Получение списка подписок на события / event_subscription_list
