BaaS для партнеров : Получение статуса перевода card2card /

Created by Наталья Боровикова, last modified by Наталья Моргунова on янв. 26, 2022

Получение статуса по переводу card2card. Параметр payment_id получается из предыдушего метода.

GET /api/v1/clients/{client_id}/card2card/{payment_id}

URL Параметры

Описание поля

Тип

payment_id

Идентификатор ссылки на пополнение

string (path)

client_id

Идентификатор клиента

string (path)



Пример ответа: Status 200 ОК


{
    "payment_id": "eykn54re",
    "is_complete": false,
    "payment_status": "in process",
    "payment_status_description": "В обработке"
}


Пример ответа ошибки токена: Status 403 Forbidden


{
    "description": "Forbidden",
    "errors": [],
    "date": "2021-10-11T08:23:48+00:00"
}


Пример ответа при несуществующем id клиента: Status 200 OK


{
    "payment_id": "eykn54re",
    "is_complete": false,
    "payment_status": "in process",
    "payment_status_description": "В обработке"
}


Пример ответа при несуществующем идентификаторе: Status 404 Not Found


{
    "description": "Requested object is not found"
}


Пример ответа при ошибке метода: Status 500 Internal Server Error


{
    "description": "Runtime error. Please try later."
}
