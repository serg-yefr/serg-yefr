BaaS для партнеров : Получение реквизитов карты / card_details

Created by Наталья Боровикова, last modified by Наталья Моргунова on нояб. 29, 2021

Метод возвращает реквизиты карты для пополнения по реквизитам (пополнение осуществляется по юридическому лицу).

GET /api/v1/clients/{client_id}/cards/{barcode}

URL параметры

Описание поля

Тип

Пример

client_id

Идентификатор клиента

string (path)

client1

barcode

Штриховой код карты

string

2000000000001

Пример ответа: Status 200 OK

{
    "card_number": "4314 0901 20** ****",
    "expiration": "11/24",
    "status": "ValidCard",
    "currency": "RUB",
    "recipient": "Иванов Иван Иванович",
    "checking_account": "40914810010550207764",
    "bank_recipient": "ТКБ БАНК ПАО",
    "legal_address": "109147, г. Москва, ул. Воронцовская, д.27/35, к/с: № 30101810800000000388 в ГУ Банка России по ЦФО",
    "bik": "044525212",
    "inn": 77123456789
}

Пример ответа ошибки токена: Status 403 Forbidden

{
    "description": "Forbidden",
    "errors": [],
    "date": "2021-10-11T08:23:48+00:00"
}

Пример ответа при ошибке метода: Status 500 Internal Server Error

{
    "description": "Runtime error. Please try later."
}
